Script WordCloud

Génération via PALS ou non

Pour générer nos nuages en fonction des résultats du script PALS, on a créé un tout petit front-end au module python wordcloud. Il est disponible ici: cloudy.py.

Qui dit Python en 2024, dit environnement virtuel. On va donc avoir besoin de créer/activer/remplir notre venv avec tous les modules dont on va avoir besoin (en gros, wordcloud & NLTK). 

python3.12 -m venv .venv
source .venv/bin/activate
pip install -U -r requirements.txt

On utilise NLTK pour récupérer une liste de mot à exclure adaptée à la langue (les stopwords).

On va ensuite tout simplement parser nos résultats PALS pour construire un dictionaire token: frequence, et, après avoir instantié un objet WordCloud, on peut tout simplement utiliser la méthode generate_from_frequencies, au lieu de la méthode generate, afin d'utiliser nos résultats au lieu d'avoir recours à l'algorithme interne du module WordCloud.

Histoire d'apprécier les différences entre ces deux approches, vous trouverez pour chaque langue deux nuages différents: le premier est généré à partir des résultats PALS, le second via l'algorithme interne de WordCloud.
